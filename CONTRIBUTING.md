# Contributing to AetherOS

感谢你对 AetherOS 的兴趣！本文档说明如何参与项目。

## 🚀 开始参与

### 第一步：了解项目

1. 阅读 [项目概览](docs/PROJECT.md)
2. 查看 [治理框架](docs/GOVERNANCE.md)
3. 浏览 [现有 Discussions](../../discussions)

### 第二步：选择参与方式

#### 💡 分享想法
- 在 [Discussions](../../discussions) 中提出新想法
- 标记为 `idea` 或 `design`
- 等待社区反馈

#### 📝 提交 RFC
- 当想法获得支持后，可以撰写 RFC
- 遵循 [RFC 流程](docs/rfcs/README.md)
- 创建 PR 并收集反馈

#### 🛠️ 参与实施
- 查看 [项目看板](../../projects)
- 认领 Issue 或创建新 Issue
- 提交 Pull Request

#### 📖 改进文档
- 修复错别字或不清楚的说明
- 添加示例或教程
- 改进现有文档

## 📋 提交 Issue

创建 Issue 时：

1. **清晰的标题** - 准确描述问题或建议
2. **详细描述** - 提供足够的上下文
3. **相关链接** - 链接相关的 Discussion、RFC 或 ADR
4. **标签** - 使用适当的标签进行分类

## 🔄 提交 Pull Request

### 步骤

1. **Fork 仓库**
   ```bash
   git clone https://github.com/xukin10/AetherOS.git
   cd AetherOS
   ```

2. **创建特性分支**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **提交更改**
   ```bash
   git add .
   git commit -m "feat: clear description of changes"
   ```

4. **推送到 Fork**
   ```bash
   git push origin feature/your-feature-name
   ```

5. **创建 Pull Request**
   - 提供清晰的 PR 描述
   - 引用相关的 Issue 或 Discussion
   - 等待审查

### PR 清单

- [ ] 标题清晰简洁
- [ ] 描述包含足够上下文
- [ ] 链接相关 Issue/Discussion
- [ ] 添加适当的标签
- [ ] 通过本地测试（如适用）

## 📚 RFC 流程

对于重大更改，请遵循 RFC 流程：

1. 在 Discussions 中讨论你的想法
2. 使用 [RFC 模板](docs/rfcs/RFC-TEMPLATE.md) 撰写 RFC
3. 创建 PR，标记为 `type:rfc`
4. 收集反馈并迭代
5. 获得批准后创建 ADR

详见 [RFC 流程指南](docs/rfcs/README.md)

## ✅ 行为准则

### 我们的承诺

- 尊重所有贡献者
- 欢迎不同的观点
- 提供建设性的反馈
- 创建包容的社区

### 不可接受的行为

- 骚扰、欺凌或歧视
- 个人攻击或辱骂
- 骚扰或威胁
- 发布他人私人信息

任何违反行为准则的行为将被举报和处理。
